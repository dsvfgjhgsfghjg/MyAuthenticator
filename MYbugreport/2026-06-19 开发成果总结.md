## 🧩 核心变更详解

### 1️⃣ 设备密钥流迁移（Server-Driven Device Secret）

**目标：** 从客户端自行生成设备 ID 的模式，迁移到由服务端统一签发 `deviceId` + `deviceSecret` 的安全方案。

#### 涉及文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `DeviceIdManager.kt` | 🔄 重构增强 | 新增 `ensureDeviceSecret()` suspend 函数，调用服务端注册接口获取 `svr_<UUID>` 格式 deviceId + 64 位 hex secret |
| `SecureStorage.kt` | 🔄 重构增强 | 新增 `deviceId` / `deviceSecret` 存储字段，保留向后兼容的旧字段 |
| `BindRequest.kt` | ✏️ 修改 | 请求体新增 `deviceSecret` 字段 |
| `WebSocketMessage.kt` | ✏️ 修改 | Bind 消息模型新增 `deviceSecret` 字段 |
| `ApiService.kt` | ➕ 新增接口 | 新增 `registerDeviceSecret()` API 调用（与服务端设备注册接口对接） |

#### 业务流程变更

```
旧流程（客户端自生成）:
  客户端 -> 生成 ANDROID_<hash>_<random>
  客户端 -> 登录/绑定时发送 deviceId
  服务端 -> 记录并返回 token

新流程（服务端签发）:
  客户端 -> ensureDeviceSecret() -> 调用服务端注册接口
  服务端 -> 返回 svr_<UUID> + 64位hex deviceSecret
  客户端 -> 本地持久化存储
  客户端 -> WebSocket bind 时携带 deviceSecret 供服务端验证
```

- 发送 bind 消息时，新增 `deviceSecret` 字段传递给服务端
- WebSocketService 保存 `savedDeviceSecret` 状态，支持重连时恢复

### 4️⃣ 主界面流程重构（MainActivity）

#### 涉及文件

| `MainActivity.kt` | 🔄 重构 | 移除启动时自动跳转登录的逻辑，改为用户主动触发 |

#### 关键修改

| 修改项 | 旧行为 | 新行为 |
|--------|--------|--------|
| 登录判断 | 启动时检查 `isBound`，未绑定跳转 | 启动时调用 `ensureDeviceSecret()` 确保设备 ID 存在 |
| 登录状态 | 启动时无条件跳转 LoginActivity | `isLoggedIn()` 检查 JWT + deviceId |
| WebSocket 启动 | 每次启动无条件启动 | 仅当 `isLoggedIn() == true` 时启动 |
| Challenge 弹窗 | 使用 `currentChallengeDialog` 字段追踪 | 通过 FragmentManager tag 检查唯一性 |
| 认证结果展示 | 零散 Toast 逻辑 | 统一 `getAuthResultMessage()` 方法 |
